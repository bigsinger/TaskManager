# TaskManager v1.2 CSRF 防护设计文档

## 功能概述

实现 CSRF（Cross-Site Request Forgery）防护，防止跨站请求伪造攻击。

## 技术方案

### 1. CSRF Token 生成

使用 crypto 库生成随机的 CSRF Token。

### 2. CSRF Token 存储

将 CSRF Token 存储在 Session 中，并通过 Cookie 返回给客户端。

### 3. CSRF Token 验证

在 POST、PUT、DELETE 请求中验证 CSRF Token。

## 实现步骤

### 1. 安装依赖

```bash
npm install csurf cookie-parser
npm install --save-dev @types/csurf @types/cookie-parser
```

### 2. 配置 CSRF 中间件

```typescript
import cookieParser from 'cookie-parser';
import csrf from 'csurf';

// Cookie parser
app.use(cookieParser());

// CSRF protection
const csrfProtection = csrf({
  cookie: {
    httpOnly: true,
    secure: process.env.NODE_ENV === 'production',
    sameSite: 'strict'
  }
});

// Get CSRF token
app.get('/api/csrf-token', csrfProtection, (req, res) => {
  res.json({
    csrfToken: req.csrfToken()
  });
});

// Apply CSRF protection to state-changing requests
app.use('/api', csrfProtection);
```

### 3. 前端集成

```javascript
// Get CSRF token
async function getCsrfToken() {
  const response = await fetch('/api/csrf-token');
  const data = await response.json();
  return data.csrfToken;
}

// Add CSRF token to requests
const originalFetch = window.fetch;
window.fetch = function(url, options = {}) {
  if (options.method && ['POST', 'PUT', 'DELETE'].includes(options.method.toUpperCase())) {
    const csrfToken = document.querySelector('meta[name="csrf-token"]')?.getAttribute('content');
    if (csrfToken) {
      options.headers = options.headers || {};
      options.headers['X-CSRF-Token'] = csrfToken;
    }
  }
  return originalFetch(url, options);
};

// Initialize CSRF token
document.addEventListener('DOMContentLoaded', async () => {
  const csrfToken = await getCsrfToken();
  const meta = document.createElement('meta');
  meta.name = 'csrf-token';
  meta.content = csrfToken;
  document.head.appendChild(meta);
});
```

### 4. HTML 更新

```html
<head>
  <meta name="csrf-token" content="">
</head>
```

## 安全考虑

1. **HttpOnly Cookie**：防止 JavaScript 访问 Cookie
2. **Secure Cookie**：只在 HTTPS 下传输
3. **SameSite Cookie**：防止跨站请求
4. **Token 验证**：验证所有状态改变请求

## 测试

### 1. 正常请求测试

- ✅ 带有有效 CSRF Token 的请求应该成功
- ✅ GET 请求不需要 CSRF Token

### 2. 攻击测试

- ❌ 没有 CSRF Token 的请求应该失败
- ❌ 无效的 CSRF Token 应该失败
- ❌ 过期的 CSRF Token 应该失败
