# TaskManager v2.1 设计方案

## 版本目标
实现多租户、组织架构、权限管理和任务评论功能的企业级任务管理系统。

---

## 一、多租户架构

### 1.1 核心概念
- **Tenant（租户）**: 代表一家公司/组织
- 每个租户数据完全隔离
- 通过子域名或路径区分租户（如：company1.taskmanager.com）

### 1.2 数据库设计
```sql
-- 租户表
tenants:
  - id: UUID PRIMARY KEY
  - name: VARCHAR(100) 公司名称
  - subdomain: VARCHAR(50) UNIQUE 子域名
  - status: ENUM('active', 'suspended', 'deleted')
  - created_at, updated_at

-- 用户表增加tenant_id
tenants_users:
  - id: UUID PRIMARY KEY
  - tenant_id: UUID FK → tenants.id
  - email: VARCHAR(100) UNIQUE
  - name: VARCHAR(100)
  - password_hash: VARCHAR(255)
  - role: ENUM('owner', 'admin', 'user')
  - status: ENUM('active', 'inactive')
  - created_at, updated_at
```

## 二、组织架构（三级Group）

### 2.1 层级结构
```
租户 (Tenant)
├── 一级部门 (Department)
│   ├── 二级团队 (Team)
│   │   ├── 三级小组 (Group)
│   │   └── 三级小组 (Group)
│   └── 二级团队 (Team)
└── 一级部门 (Department)
    └── ...
```

### 2.2 数据库设计
```sql
-- 组织架构表
groups:
  - id: UUID PRIMARY KEY
  - tenant_id: UUID FK → tenants.id
  - parent_id: UUID FK → groups.id (nullable, 顶级为null)
  - name: VARCHAR(100) 组织名称
  - level: TINYINT 层级 (1, 2, 3)
  - type: ENUM('department', 'team', 'squad') 类型
  - manager_id: UUID FK → tenants_users.id 负责人
  - created_at, updated_at

-- 用户-组织关联表
group_members:
  - id: UUID PRIMARY KEY
  - group_id: UUID FK → groups.id
  - user_id: UUID FK → tenants_users.id
  - joined_at: TIMESTAMP
  - UNIQUE(group_id, user_id)
```

## 三、权限系统

### 3.1 角色设计
| 角色 | 英文 | 权限范围 |
|------|------|----------|
| 拥有者 | Owner | 全部权限（创建租户时自动分配） |
| 任务执行者 | TaskExecutor | 只能修改自己任务的状态 |
| 项目管理者 | ProjectManager | 可修改/变更状态，不可修改任务内容，可追加评论 |

### 3.2 权限表设计
```sql
-- 角色表
roles:
  - id: UUID PRIMARY KEY
  - tenant_id: UUID FK → tenants.id
  - name: VARCHAR(50) 角色名
  - permissions: JSON 权限配置
  - is_system: BOOLEAN 是否系统预设

-- 用户角色表（一个用户可有多个角色）
user_roles:
  - id: UUID PRIMARY KEY
  - user_id: UUID FK → tenants_users.id
  - role_id: UUID FK → roles.id
  - group_id: UUID FK → groups.id (nullable, 全局角色为null)

-- 权限常量
PERMISSIONS = {
  TASK_CREATE: 'task:create',
  TASK_EDIT: 'task:edit',
  TASK_DELETE: 'task:delete',
  TASK_STATUS_CHANGE: 'task:status:change',
  TASK_COMMENT: 'task:comment',
  TASK_ASSIGN: 'task:assign',
  ROLE_CREATE: 'role:create',
  ROLE_ASSIGN: 'role:assign',
  GROUP_CREATE: 'group:create',
  GROUP_MANAGE: 'group:manage'
}
```

### 3.3 默认角色权限
```json
{
  "Owner": {
    "permissions": ["*"]
  },
  "TaskExecutor": {
    "permissions": [
      "task:status:change:own",
      "task:comment"
    ]
  },
  "ProjectManager": {
    "permissions": [
      "task:status:change",
      "task:comment",
      "task:assign"
    ]
  }
}
```

## 四、任务评论系统

### 4.1 功能特性
- 支持富文本编辑（Markdown/HTML）
- 支持附件上传
- 支持@提及用户
- 支持子任务拆分标记
- 按时间倒序展示

### 4.2 数据库设计
```sql
-- 任务评论表
task_comments:
  - id: UUID PRIMARY KEY
  - task_id: UUID FK → tasks.id
  - author_id: UUID FK → tenants_users.id
  - parent_id: UUID FK → task_comments.id (回复功能，可选)
  - content: TEXT 评论内容（富文本）
  - content_type: ENUM('text', 'markdown', 'html')
  - attachments: JSON 附件列表
  - mentions: JSON @提及用户列表
  - is_system: BOOLEAN 是否系统生成（如状态变更记录）
  - created_at, updated_at

-- 评论附件表
comment_attachments:
  - id: UUID PRIMARY KEY
  - comment_id: UUID FK → task_comments.id
  - file_name: VARCHAR(255)
  - file_path: VARCHAR(500)
  - file_size: INT
  - mime_type: VARCHAR(100)
  - uploaded_by: UUID FK → tenants_users.id
  - uploaded_at: TIMESTAMP
```

## 五、任务默认排序

### 5.1 排序规则
1. **主排序**: 状态优先级（未开始 > 进行中 > 已完成）
2. **次排序**: 创建时间倒序（最新的在前）

### 5.2 状态优先级映射
```javascript
const STATUS_PRIORITY = {
  'pending': 1,      // 未开始
  'in-progress': 2,  // 进行中
  'completed': 3     // 已完成
};

// SQL排序
ORDER BY 
  CASE status 
    WHEN 'pending' THEN 1 
    WHEN 'in-progress' THEN 2 
    WHEN 'completed' THEN 3 
  END ASC,
  created_at DESC
```

## 八、API设计概览

### 租户管理
```
POST   /api/tenants              创建租户
GET    /api/tenants/:id          获取租户信息
PUT    /api/tenants/:id          更新租户
DELETE /api/tenants/:id          删除租户
```

### 组织架构
```
POST   /api/groups               创建组织
GET    /api/groups               获取组织列表（树形）
GET    /api/groups/:id           获取组织详情
PUT    /api/groups/:id           更新组织
DELETE /api/groups/:id           删除组织
POST   /api/groups/:id/members   添加成员
DELETE /api/groups/:id/members/:userId 移除成员
```

### 角色权限
```
GET    /api/roles                获取角色列表
POST   /api/roles                创建角色
PUT    /api/roles/:id            更新角色
DELETE /api/roles/:id            删除角色
POST   /api/users/:id/roles      分配角色
```

### 任务评论
```
GET    /api/tasks/:id/comments   获取评论列表
POST   /api/tasks/:id/comments   创建评论
PUT    /api/comments/:id         更新评论
DELETE /api/comments/:id         删除评论
POST   /api/comments/:id/attachments 上传附件
```
