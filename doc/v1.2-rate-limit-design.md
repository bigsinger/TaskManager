# TaskManager v1.2 速率限制设计文档

## 功能概述

实现细粒度的速率限制，防止暴力攻击和滥用。

## 技术方案

### 1. 速率限制策略

#### 1.1 全局限速

限制所有 API 请求的速率。

#### 1.2 认证限速

限制认证相关 API 的速率。

#### 1.3 IP 限速

限制单个 IP 的请求速率。

#### 1.4 用户限速

限制单个用户的请求速率。

### 2. 速率限制配置

```typescript
// 全局限速
const globalLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 分钟
  max: 100, // 最多 100 个请求
  message: 'Too many requests, please try again later'
});

// 认证限速
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 分钟
  max: 5, // 最多 5 个请求
  message: 'Too many authentication attempts, please try again later'
});

// IP 限速
const ipLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 分钟
  max: 50, // 最多 50 个请求
  keyGenerator: (req) => req.ip,
  message: 'Too many requests from this IP, please try again later'
});

// 用户限速
const userLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 分钟
  max: 30, // 最多 30 个请求
  keyGenerator: (req) => req.user?.userId || req.ip,
  message: 'Too many requests, please try again later'
});
```

### 3. 应用速率限制

```typescript
// 应用全局限速
app.use('/api', globalLimiter);

// 应用认证限速
app.use('/api/auth', authLimiter);

// 应用 IP 限速
app.use('/api', ipLimiter);

// 应用用户限速（需要认证）
app.use('/api', authMiddleware, userLimiter);
```

## 实现步骤

### 1. 创建速率限制配置文件

```typescript
// src/backend/src/infrastructure/config/rateLimitConfig.ts

export const rateLimitConfig = {
  global: {
    windowMs: 60 * 1000,
    max: 100,
    message: 'Too many requests, please try again later'
  },
  auth: {
    windowMs: 15 * 60 * 1000,
    max: 5,
    message: 'Too many authentication attempts, please try again later'
  },
  ip: {
    windowMs: 60 * 1000,
    max: 50,
    message: 'Too many requests from this IP, please try again later'
  },
  user: {
    windowMs: 60 * 1000,
    max: 30,
    message: 'Too many requests, please try again later'
  }
};
```

### 2. 创建速率限制中间件

```typescript
// src/backend/src/infrastructure/middleware/rateLimit.ts

import rateLimit from 'express-rate-limit';
import { rateLimitConfig } from '../config/rateLimitConfig';

export const globalLimiter = rateLimit(rateLimitConfig.global);
export const authLimiter = rateLimit(rateLimitConfig.auth);
export const ipLimiter = rateLimit({
  ...rateLimitConfig.ip,
  keyGenerator: (req) => req.ip
});
export const userLimiter = rateLimit({
  ...rateLimitConfig.user,
  keyGenerator: (req) => req.user?.userId || req.ip
});
```

### 3. 应用速率限制

```typescript
// src/backend/src/server.ts

import { globalLimiter, authLimiter, ipLimiter, userLimiter } from './infrastructure/middleware/rateLimit';

// 应用全局限速
app.use('/api', globalLimiter);

// 应用认证限速
app.use('/api/auth', authLimiter);

// 应用 IP 限速
app.use('/api', ipLimiter);

// 应用用户限速（需要认证）
app.use('/api', authMiddleware, userLimiter);
```

## 安全考虑

1. **防止暴力攻击**：限制认证请求的速率
2. **防止滥用**：限制单个用户的请求速率
3. **防止 DDoS**：限制单个 IP 的请求速率
4. **灵活配置**：支持不同端点的不同限速策略
