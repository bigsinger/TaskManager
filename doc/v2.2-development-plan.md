## 2. v2.2 新增需求

### 2.1 情境（Context）管理
**需求**:
- 创建contexts表，用于管理项目/情境
- 每个任务属于一个情境
- 支持情境的CRUD操作
- 在任务列表中按情境筛选

**实现**:
```javascript
// 数据库表
CREATE TABLE IF NOT EXISTS contexts (
  id TEXT PRIMARY KEY,
  tenant_id TEXT NOT NULL,
  name TEXT NOT NULL,
  description TEXT,
  avatar TEXT,
  owner_id TEXT NOT NULL,
  is_public INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (tenant_id) REFERENCES tenants(id) ON DELETE CASCADE,
  FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE
)

// 任务表添加情境ID
ALTER TABLE tasks ADD COLUMN context_id TEXT;
```

### 2.2 任务角色扩展
**需求**:
- 任务表添加报告人(reporter_id)和验证人(verifier_id)字段
- 在任务详情页显示三种角色：执行者、报告人、验证人

**实现**:
```javascript
// 任务表添加字段
ALTER TABLE tasks ADD COLUMN reporter_id TEXT;
ALTER TABLE tasks ADD COLUMN verifier_id TEXT;
```

### 2.3 任务预计耗时
**需求**:
- 任务表添加estimated_time字段（JSON格式）
- 支持时间单位：h/d/w/mo/q/y
- 在任务详情页显示

**实现**:
```javascript
// 任务表添加字段
ALTER TABLE tasks ADD COLUMN estimated_time TEXT; -- JSON格式: {value, unit, display}
```

### 2.4 任务收藏功能
**需求**:
- 任务表添加is_favorite字段
- 在任务列表中显示收藏按钮
- 支持筛选收藏的任务

**实现**:
```javascript
// 任务表添加字段
ALTER TABLE tasks ADD COLUMN is_favorite INTEGER DEFAULT 0;
```

### 2.5 用户信息扩展
**需求**:
- 用户表添加nickname和description字段
- 支持编辑用户信息
- 在个人中心显示

**实现**:
```javascript
// 用户表添加字段
ALTER TABLE users ADD COLUMN nickname TEXT;
ALTER TABLE users ADD COLUMN description TEXT;
```

### 2.6 时间线功能
**需求**:
- 增强现有的task_activities表
- 支持更多操作类型（create/start/comment/edit/status_change/complete/assign/favorite）
- 前端显示时间线

**实现**:
- 后端：使用现有的task_activities表
- 前端：创建Timeline组件

### 2.7 手势交互
**需求**:
- 在任务卡片上支持手势操作
- 向右滑动：切换任务状态
- 向左滑动：显示时间线
- 支持移动端触摸和PC端鼠标拖拽

**实现**:
- 前端：使用useSwipeGesture hook
- 动画：使用CSS transitions

### 2.8 情境切换器
**需求**:
- 在任务列表页顶部显示情境切换器
- 支持快速切换当前情境
- 高亮当前激活的情境

**实现**:
- 前端：ContextSwitcher组件
- API：获取用户的情境列表

### 2.9 用户头像组件
**需求**:
- 显示用户头像和昵称
- 点击头像进入个人中心
- 支持第三方登录的昵称

**实现**:
- 前端：UserAvatar组件
- API：获取和更新用户信息

### 2.10 个人中心
**需求**:
- 个人中心页面
- 支持编辑昵称、描述
- 显示组织架构、角色信息、统计数据

**实现**:

## 5. API设计

### 5.1 情境API
- `GET /api/contexts` - 获取用户的所有情境
- `POST /api/contexts` - 创建情境
- `PUT /api/contexts/:id` - 更新情境
- `DELETE /api/contexts/:id` - 删除情境

### 5.2 用户API
- `GET /api/users/profile` - 获取当前用户信息
- `PUT /api/users/profile` - 更新用户信息
- `GET /api/users/:id/organizations` - 获取用户所属组织
- `GET /api/users/:id/roles` - 获取用户的角色
- `GET /api/users/:id/stats` - 获取用户统计数据

### 5.3 任务API
- `GET /api/tasks?context_id=xxx&is_favorite=1` - 获取任务列表（支持筛选）
- `POST /api/tasks` - 创建任务
- `PUT /api/tasks/:id` - 更新任务
- `PATCH /api/tasks/:id/favorite` - 切换收藏状态

### 5.4 活动日志API
- `GET /api/tasks/:id/activities` - 获取任务活动日志
- `POST /api/tasks/:id/activities` - 创建活动日志
